# 📘 1일차 — DB 기초 & SQL SELECT

## 1. 데이터베이스(DB) 기초

**정의** : 유용한 데이터를 검색·수정·삭제가 용이하도록 저장한 집합

| 구분 | 설명 |
|------|------|
| 통합 데이터 | 여러 부서 데이터를 한 곳에서 공동 관리, 중복 최소화 |
| 저장 데이터 | 디스크 등 저장 매체에 보관 |
| 운영 데이터 | 조직 운영에 필수적인 데이터 |
| 공용 데이터 | 여러 사용자가 서로 다른 목적으로 공유 |

**특징** : 실시간 접근성 / 계속적 변화 / 동시 공유 / 내용에 의한 참조

---

## 2. DBMS

> 응용프로그램과 데이터 사이의 **중재자**. DBMS를 통해서만 데이터 접근 가능.

| 기능 | 설명 |
|------|------|
| 정의 기능 | 앱-DB 간 인터페이스 방법 제공 |
| 조작 기능 | 검색·삽입·삭제·갱신 |
| 제어 기능 | 무결성·보안·병행제어 |

- ✅ **장점** : 중복 최소화, 일관성·무결성·보안 유지, 표준화
- ❌ **단점** : 운영비 증대, 복잡한 백업/회복, 시스템 취약성

## 3. 데이터베이스 언어

| 언어 | 풀네임 | 역할 |
|------|--------|------|
| DDL | Data Definition Language | 스키마 정의 |
| DML | Data Manipulation Language | 데이터 조작 (검색·삽입·삭제·변경) |
| DCL | Data Control Language | 보안·무결성·병행제어 |

---

## 4. 관계형 DB 핵심 용어

| 공식 용어 | 일반 용어 | 설명 |
|-----------|-----------|------|
| 릴레이션 | 테이블 | 행과 열로 구성된 기본 저장 단위 |
| 튜플 | 행(Row) | 하나의 레코드 |
| 애트리뷰트 | 열(Column) | 단일 종류의 데이터 |
| 도메인 | - | 한 열에 올 수 있는 값들의 집합 |
| 차수(Degree) | - | 열의 수 |
| 카디날리티 | - | 행의 수 |

---

## 5. 키(Key) 종류

| 키 | 설명 |
|----|------|
| 슈퍼 키 | 튜플을 고유하게 식별하는 열들의 집합 |
| 후보 키 | 고유 식별하는 **최소** 열들의 집합 |
| 기본 키(PK) | 후보 키 중 선정 — NULL 불가, 유일 |
| 대체 키 | 기본 키가 아닌 후보 키 |
| 외래 키(FK) | 다른 테이블의 PK를 참조 → 테이블 간 관계 표현 |

---

## 6. 무결성 제약조건

| 제약 | 설명 |
|------|------|
| 도메인 제약 | 각 열 값은 원자값이어야 함 |
| 키 제약 | 키 열에 중복 값 불가 |
| 엔티티 무결성 | PK는 NULL 불가, 유일해야 함 |
| 참조 무결성 | FK는 반드시 존재하는 PK 값을 참조해야 함 |

---

## 7. SQL 기초 — SELECT

```sql
SELECT [DISTINCT] 컬럼명 | *
FROM   테이블명
WHERE  조건
ORDER BY 컬럼 [ASC | DESC];
```

| 키워드 | 역할 |
|--------|------|
| `DISTINCT` | 중복 행 제거 |
| `*` | 모든 열 출력 |
| `WHERE` | 조건 필터링 |
| `ORDER BY` | 정렬 (기본값 ASC) |

> 💡 SQL은 대소문자 구분 없음. 키워드는 대문자, 테이블/컬럼명은 소문자 권장. 문장 끝에 `;` 필수.
